# 面试题答案

## 模块一

**面试题：一个 session 连续聊了 200 轮，context 快爆了。你会怎么做压缩？如何确保压缩后的对话仍然流畅？**

**回答：** 关于压缩的部分先分成三层，分为短期记忆中期记忆和长期记忆，短期记忆通过滚动对话，就是当前上下文的对话中获得。从短期记忆变成中期记忆，可能通过几个方式，包括固定设计一个时间，执行压缩，或者是上下文快满了自动压缩，完成一个任务进行压缩，压缩时保留对话中的几类主要信息，如用户偏好，任务进度，决策习惯，和提到的实体等等，中期的记忆比起长期记忆可以多详细一点因为用户使用的概率相对长期记忆会更高。中期记忆过一段固定时间进一步压缩保存为长期记忆，进一步提取重点节省存储空间等待用户召回使用。在上下文满了之后执行第一次压缩的时候主要是省略掉对任务帮助不大的东西，同时把当前滚动记忆进行任务拆分分别保存，同时引入任务卡概括任务主要信息如进度等，保留滚动记忆确保之后对话流畅，然后等待下一次对话读取到相应任务卡的记忆，保证下次继续任务时候上下文有足够的容量进行，并且对任务的信息丢失较少。

## 模块二

**面试题：你理解的 Agent memory 经典框架是什么？它的发展趋势是什么，最头部的玩家在怎么做？**

**回答：** 记忆的经典框架分为三层，像模块一回答的，分为短期中期长期记忆，短期就是滑动窗口的记忆，中期是经过压缩的记忆，长期记忆通过向量化存入向量库，我认为的发展趋势是把memory作为独立服务迭代更新，从刚开始的只有短期记忆发展到现在的按照需要召回中长期记忆。头部的agent我了解了几个如memgpt，通过调用api能力让大模型自己进行记忆管理，记忆迁移，openai的则是偏向rag路线，搜索文件+记忆。

## 模块三

**面试题：对于长程任务，大模型执行一段时间可能会忘掉目标，你知道哪些解决方案，有什么优缺？**

**回答：**

**方案一：** 可以通过引入任务卡，在任务进程中时把目标进度和约束等写成结构化卡片，在过程中时刻更新并在进行任务时候读取。
- 优点：可以复用进度不会丢
- 缺点：需要设计好schema

**方案二：** 可以在每n步让大模型自检任务，确保任务还在正常进行中没有出现错误，有点像react架构。
- 优点：可以动态纠偏
- 缺点：会有额外token花销

## 模块四

**面试题：Agent 工具有同步和异步两类。异步工具不能让用户一直等，但结果依然重要。你会如何设计异步工具执行和完成通知？**

**回答：** 异步工具的核心思路是为每一个用户需求创造一个id，然后agent在不影响主要进程的前提下后台执行，完成后根据id返回执行结果。通知方式分为轮询，主动推送和webhook，轮询是用户主动询问，主动推送是agent在task完成后推送到用户的对话框，webhook则是在主动推送的基础上将执行结果推送到指定的url里让用户知道。在异步工具使用时候应该注重和用户的交互，比如在busy状态时候让用户知道，告诉用户正在处理中。我会设计成向用户发送taskid和预计完成时间，让用户有预期同时在完成时候主动将结果push到用户能收到的地方，达到效率最大化。

## 模块五

**面试题：Claude Code 的工具输出方式和国内 GLM / 豆包等 OpenAI-compatible function calling 有什么不同？他们各自这样设计的优缺点是什么？**

**回答：**

Claude的工具输出是xml嵌入文本式，其他一般是使用json。

**Claude XML 方式的优势：**
- xml比起json更加灵活
- 可以将思考和行动相结合，对工具的调用穿插在文字里
- 更加灵活的输出可以让大模型有修补错误的空间，没有将工具写死
- 用户在调试看日志的时候面对的文件也更加具有可读性

**Claude XML 方式的缺点：**
- 更加灵活的文本也使得大模型在处理相关工作时比起结构固定的json会话花费较多时间

**JSON 方式的逻辑：**
- 兼容性强工程化简单约束性强
- 在工业生产环境时更加稳定可靠
- 缺点相对应的就是不灵活可读性差
