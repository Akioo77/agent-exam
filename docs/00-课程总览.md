## 🎯 学习顺序建议

**两轮学习法**：
- **第一轮（快速通读，1-2 小时）**：先看每课的"核心概念清单"和"考点点"，知道每个模块在讲什么
- **第二轮（深度学习，2-3 小时）**：挑和题目最相关的章节细读

**优先级**（先学这些，因为直接对应题目）：
1. **01-Agent-Runtime 基础** ⭐⭐⭐（Vibe coding 核心）
2. **02-Context-Engineering** ⭐⭐⭐（题目模块一）
3. **05-Tool-Runtime** ⭐⭐（题目模块四）
4. **03-Memory 框架** ⭐⭐（题目模块二）
5. **04-Task Management** ⭐（题目模块三）
6. **06-架构对比** ⭐（题目模块五，最容易临场发挥）

## 💡 学习方法

每篇课程的结构：
```
1. 核心概念清单（这是什么？）
2. 原理与代码示例（怎么工作？）
3. 实战要点（生产环境怎么用？）
4. 题目考点（笔试会怎么考？）
5. 推荐阅读（深入学习）
```

**不要死记硬背**——题目考的是"思考过程"和"trade-off 意识"。
